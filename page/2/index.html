发表于 2019-08-23 | 更新于 2019-09-30 | 分类于 redis

数据类型以及应用场景

Redis不仅是一个单纯的K-V存储,还是一个数据结构服务器.支持5种数据类型:

  1. 字符串String,可以包含任何数据,二进制安全的.最大512MB.
  2. Lists: 有序列表,保持插入的顺序.可以添加元素到列表头部或者尾部.最多可以存储2^32-1个元素.
  3. Sets: 集合,无序,唯一.最多存储2^32-1个元素.
  4. Sorted sets: 有序集合.每个元素有一个分数,根据分数从小到大排序.元素唯一,但是分数可以相同.
  5. 字典Hashs: k-v都是String.一个Hash可以存储2^32-1个键值对.适合存储对象.
  6. HyperLogLog: 基数统计, 在输入元素的数量或者体积非常非常大的时候,计算基数所需空间总是固定的,很小的.基数估计就是在误差范围可接受的情况下快速计算基数(不重复元素).典型应用场景是计算UV, IP数等指标.
  7. Bit Arrays: 位数组.
  8. Streams: 类似消息队列.
  9. GEO: 地理位置信息.典型应用是获取附近x公里的车辆,附近商家等.

JVM是什么

Java虚拟机是一台由规范来定义的抽象计算机,其规范定义了每个JVM都必须实现的特性,允许特定的实现使用不同的技术.当运行一个Java程序的同时,也就在运行一个JVM实例.

JVM就是为了负责运行Java程序而生.启动Java程序JVM也就开始了,退出程序JVM就结束了.每个Java程序都运行在自己的JVM中.

JVM的内部体系结构主要包含: 类装载子系统, 运行时数据区(方法区, 堆, Java栈, 程序计数器, 本地方法栈), 执行引擎,JIT编译器,垃圾收集器,本地方法接口.

Java通过使用JVM屏蔽了与具体平台相关的信息,使得Java具备了一次编写,多处运行的特性.

第三方支付

第三方(没有资金所有权,只中转)支付(需要申请支付业务许可证): 完成货币支付, 资金清算, 查询统计等系列过程.

为什么需要第三方支付?
  • 屏蔽银行支付接口的差异,对外提供统一接入平台,类似中间件.
  • 安全等级和易用性之间平衡,方便用户,提高体验.
  • 大型企业(阿里,腾讯)信用背书.

分布式系统开发

什么是分布式系统?

第一映像: 不同地点, 不同规模, 不同目的, 彼此联系, 复杂, 异构系统.

横看成岭侧成峰,远近高低各不同

常见误区

  1. 分布式系统 == SOA/ESB/微服务

    服务化的本质是”分而治之”.分治的手段就是拆分,拆成高内聚,低耦合的独立模块组以降低协作复杂度.怎么拆的具体手段很多,SOA/ESB/MS等并不是关键.关键的是拆分的原则(怎么梳理,归类,封装).

  2. 分布式系统 ? 中间件(总会提到各种中间件的名字)

    中间件的本质是”标准化”.中间件是为了解决一类问题(或者解决差异化)的工具.比如MQ解决异步通信,RPC解决实时通信,DAL统一数据访问.分布式系统会用到各种中间件.

  3. 单系统 + 单数据库 != 分布式系统

    单系统单db也是多个进程协作完成一个功能.
