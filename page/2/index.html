分布式系统开发

什么是分布式系统?

第一映像: 不同地点, 不同规模, 不同目的, 彼此联系, 复杂, 异构系统.

横看成岭侧成峰,远近高低各不同

常见误区

  1. 分布式系统 == SOA/ESB/微服务

    服务化的本质是”分而治之”.分治的手段就是拆分,拆成高内聚,低耦合的独立模块组以降低协作复杂度.怎么拆的具体手段很多,SOA/ESB/MS等并不是关键.关键的是拆分的原则(怎么梳理,归类,封装).

  2. 分布式系统 ? 中间件(总会提到各种中间件的名字)

    中间件的本质是”标准化”.中间件是为了解决一类问题(或者解决差异化)的工具.比如MQ解决异步通信,RPC解决实时通信,DAL统一数据访问.

  3. 单系统 + 单数据库 != 分布式系统

    单系统单db也是多个进程协作完成一个功能.

HashMap

  1. 构造器. 默认容量16, 负载因子0.75, 当size > 16 * 0.75时自动扩容.
  2. 底层结构: 数组+链表(当链表膨胀到8并且数组长度大于64时,链表转化成红黑树)
  3. put: 首先计算hash(key), 然后判断是否需要初始化数组, 再根据(length-1)&hash获取下标,根据情况插入: 如果当前位置为null,插入;如果当前有值(发生hash碰撞),要么替换,要么放到红黑树里,要么尾插法放到链表里.
  4. get: 计算hash值, 定位, 判断第一个节点是否符合(hash相同,key相同),然后查找树或者迭代链表.
  5. 迭代: keySet, entrySet, forEach等.常用entrySet,一次就可以取出key和value.Jdk1.8里entrySet和forEach是相同的实现.原理都是外层迭代数组,内层迭代链表或者红黑树.

关系型数据库

发表于 2019-06-12 | 分类于 db

关系型数据库

常见厂商: Oracle, MySQL, PostgreSQL, SQL Server等.

常见连接方式:ODBC, OLE DB, ADO, ADO.NET, JDBC.

  • ODBC: Open Database Connectivity, 微软推出, 只针对关系型数据库.
  • OLE DB: Object Linking and Embedding Database, 微软推出,基于COM思想提供统一的数据访问接口,包括RDB, 邮件, 目录服务等.
  • ADO: ActiveX Data Objects, 微软推出,基于COM.
  • ADO.NET: 从ADO发展过来,使用.NET而不是ActiveX技术,可以离线工作,同时有ORM的功能.
  • JDBC: Java Database Connectivity, Java推出,包括两层接口:应用程序层(面向应用开发)和驱动程序层(面向具体厂商驱动).

进程和线程

进程有自己的执行环境和内存空间,通常看起来一个程序/应用就是一个进程,不过有时候一个程序也可能是由多个进程协作组成的.进程间通过IPC(进程间通信)协作.大多数JVM实现都是一个单进程.

线程是在进程的范围内存在的,一个进程至少有一个线程.线程也有自己的执行环境,但是他们是共享进程的资源(内存,文件句柄等)的.很多时候一个Java进程里面就只有一个main线程.

同步异步阻塞非阻塞

发表于 2019-05-13 | 更新于 2019-08-27 | 分类于 IO

同步&异步

同步调用会一直等待消息返回才进行下一步.异步调用则是立刻返回,不用等待消息结果,具体的消息结果需要通过通知或者回调或者轮询完成.

举例:银行排队取款(同步);柜台叫号办业务(异步,喊号是通知机制).

同步异步的关键在于:消息如何通知.

阻塞&非阻塞

阻塞调用是指调用结果返回之前,当前线程被挂起(操作系统会来叫我们的),一直处于等待状态,不能执行其他任务.

非阻塞是指调用不能得到结果之前,当前线程立刻返回.非阻塞能调高CPU的利用率,但是增加了线程切换.需要权衡.

举例: 银行办业务的等待过程中,不能干其他的事,就是阻塞;可以看书看报打电话,就是非阻塞.

阻塞非阻塞的关键在于:等待结果时的状态(能不能干其他的事情).
