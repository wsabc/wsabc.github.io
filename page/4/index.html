发表于 2019-03-30 | 更新于 2019-10-01 | 分类于 open-source

如何选择开源许可证

软件开发的过程中,不可避免的需要引用,甚至修改第三方库,然后还可能需要打包,分发软件.所以在选择第三方库的过程中,一定要注意筛选该库的开源许可协议.

下图总结自阮一峰的博客(1, 2)

创意共享许可证(Creative Commons Licenses)

一种开放式协议, 明确说明使用者不能做的事,除此以外使用者可以随意使用.

常见的6种组合:

  • 署名(by license)
  • 署名-非商业用途(by-nc license)
  • 署名-禁止演绎(by-nd license)
  • 署名-非商业用途-禁止演绎(by-nc-nd license)
  • 署名-非商业用途-相同方式共享(by-nc-sa license)
  • 署名-相同方式共享(by-sa license)

Java IO

发表于 2019-03-23 | 更新于 2019-09-30 | 分类于 IO

Java IO 是一套Java用来读写数据(输入和输出)的API.大部分程序都要处理一些输入,并由输入产生一些输出.Java为此提供了java.io包.Java.io包主要涉及文件,网络数据流,内存缓冲等的输入输出.

Java IO基础

  1. 什么是流Stream?

流从概念上来说是一个连续的数据流.你既可以从流中读取数据,也可以往流中写数据.流有方向即输入和输出.

  1. 什么是字节流?字符流?

在Java IO中流既可以是字节流(以字节为单位进行读写,InputStream,OutputStream为基类),也可以是字符流(以字符为单位进行读写,Reader,Writer是基类).

灰度发布CanaryRelease

发表于 2019-02-27 | 更新于 2019-09-30 | 分类于 canary-release

灰度发布,也叫金丝雀发布,是指使服务平滑过渡的一种发布方式.在灰度发布之上可以进行A/B测试,如果用户对B没有什么反对意见,那么就逐步扩大范围(这个过程也叫灰度放量),把所有的用户都迁移到B上来.

灰度发布能及早获得用户反馈,完善产品功能,同时可以加强与用户的互动,降低产品所影响的用户范围.

核心概念

  1. 前提是新功能的引入.
  2. 可以应用在系统的某个部分上,比如UI,或者后端某个子服务.总之是系统流程中的某个环节.
  3. 一开始是少量用户介入,也可能用户没有感知.
  4. 灰度发布的核心是如何快速的切换,灰度发布不会减少错误,但是可以减少停机时间.
