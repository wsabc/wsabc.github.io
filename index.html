HTTP协议

发表于 2019-08-26 | 更新于 2019-09-12 | 分类于 http

HTTP是最常见的应用层协议,也是WEB开发不可避免的要熟悉的知识.

HTTP简介

HTTP(超文本传输协议)是在网络上传输HTML的协议,用于浏览器和服务器之间的通信,由请求和响应组成,是一个标准的CS模型.同时HTTP是一个无状态协议.

HTTP协议通常承载于TCP协议之上,有时也承载于TLS或SSL协议层之上,这个时候HTTP就成了我们常说的HTTPS.

HTTP最广泛使用的版本是1.1;目前最新的版本是2.0.之前还有0.9,1.0.

HTTP 1.1相对于1.0最主要的变化是支持长连接(Persistent Connection:建立一次 TCP 连接就能进行多次 HTTP 通信)和请求的流水线(Pipelining:在同一条长连接上连续发出请求,而不用等待响应返回,这样可以减少延迟)处理.在一个TCP连接上可以传送多个HTTP请求和响应.减少了建立和关闭连接的消耗和延迟.在HTTP1.1中默认开启Connection:keep-alive.

HTTP2.0多路复用(多个请求可同时在一个连接上并行执行)相对于HTTP1.x的管道流水线(若干个请求排队串行化处理)来说,性能更好.

OSI (Open System Interconnection)七层模型

OSI 开放系统互联模型,描述的是网络或者电信系统功能的概念性框架.分为 7 层(自底向上):

  1. 物理层(Physical): 比特(电信号,光信号)在不同介质中的表示.具体设备:网线.
  2. 数据链路层(Data Link): 负责两个直接连接的节点之间的数据传输,以及针对物理层的纠错.分为MAC 层(媒体访问控制)和 LLC层(逻辑链路控制).具体设备:交换机.
  3. 网络层(Network): 通过不同路由分组转发.具体设备:路由器. IP 协议.
  4. 传输层(Transport): 负责传输数据以及可能的算法纠错等. TCP, UDP 协议.
  5. 会话层(Session): 建立,管理,维护会话;简化传输层操作.
  6. 表示层(Presentation): 负责数据的应用格式和网络格式之间的转换.比如加解密.
  7. 应用层(Application): 负责建立与用户直接交互的应用程序.如浏览器.

JVM是什么

Java虚拟机是一台由规范来定义的抽象计算机,其规范定义了每个JVM都必须实现的特性,允许特定的实现使用不同的技术.当运行一个Java程序的同时,也就在运行一个JVM实例.

JVM就是为了负责运行Java程序而生.启动Java程序JVM也就开始了,退出程序JVM就结束了.每个Java程序都运行在自己的JVM中.

JVM的内部体系结构主要包含: 类装载子系统, 运行时数据区(方法区, 堆, Java栈, 程序计数器, 本地方法栈), 执行引擎,JIT编译器,垃圾收集器,本地方法接口.

Java通过使用JVM屏蔽了与具体平台相关的信息,使得Java具备了一次编写,多处运行的特性.

分布式系统开发

什么是分布式系统?

第一映像: 不同地点, 不同规模, 不同目的, 彼此联系, 复杂, 异构系统.

横看成岭侧成峰,远近高低各不同

常见误区

  1. 分布式系统 == SOA/ESB/微服务

    服务化的本质是”分而治之”.分治的手段就是拆分,拆成高内聚,低耦合的独立模块组以降低协作复杂度.怎么拆的具体手段很多,SOA/ESB/MS等并不是关键.关键的是拆分的原则(怎么梳理,归类,封装).

  2. 分布式系统 ? 中间件(总会提到各种中间件的名字)

    中间件的本质是”标准化”.中间件是为了解决一类问题(或者解决差异化)的工具.比如MQ解决异步通信,RPC解决实时通信,DAL统一数据访问.

  3. 单系统 + 单数据库 != 分布式系统

    单系统单db也是多个进程协作完成一个功能.
