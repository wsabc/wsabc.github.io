软件架构是什么

软件架构就是软件的基本结构,就好比房子的骨架,对软件的基本形态,功能划分,内部联通,预期扩展都做了详细的定义.常见的5种软件架构有:

1 分层架构

最常见的结构,将软件分成若干层,每一层都有明确的分工,不需要知道其他层的细节,相互之间通过接口通信.

具体分层因人而异,主要是表示层,应用层,数据层三大层.应用层还可以细化(如业务层,服务层,持久层等).

优点: 结构简单,容易理解;可以多层协同工作;每层独立性好,易于测试.

缺点: 层层传递,不易于扩展和维护,牵一发而动全身.

现代CPU

现代CPU都是从晶圆(wafer)加工成晶粒(die),再做成CPU的.在晶圆到晶粒的制作过程中,良品率跟晶粒的大小有关系.晶粒越小良品率越高.根据晶粒的大小,有以下几种布局:

  1. 单块CPU单核心:一个物理块,包含一个晶粒,无论晶粒大小,其上面只有一个CPU核心.这种性能有限.可以多线程并发执行,使用时间片轮换或者优先级策略.有上下文切换的开销.
  2. 单块CPU多核心:一个物理块,但是根据晶粒的大小可以分为: 一个大晶粒上有多个CPU核心;多个小晶粒,每个晶粒上一个CPU核心.共享一个大晶粒的性能要好,但是成本高(因为良品率低).可以多线程并行执行.每个核心上还是并发执行.
  3. 多个CPU(单/多)核心:多个物理上(多块CPU),每个CPU都可以是单核,或者多核.

现在主流的Intel就是大晶粒多核;AMD有多个小晶粒多核.举个例子形象地解释它们的区别:

大地产商建造房子都是通透大平层(大晶粒),平层上各个商家(CPU 核心)很方便沟通,也不太堵塞.

小地产商没有能力建大平层,建几个小高楼(多个小晶粒),通过地下通道互联,容易堵死.

再小的地产商连地下通道也开发不了,只能远远的建立几个小楼,通过公交等串门互联.性能最差.

数组的优缺点

发表于 2019-08-30 | 更新于 2019-10-10 | 分类于 algo

算法扫盲

算法其实就是个思考过程.解决问题的路径有很多,找到一条相对省时省力的路径.一般使用大O表示法进行衡量.

大O符号表示的是算法的最坏情况.通常是在比较算法好坏的时候才有用.但是数据量的大小对算法的实际结果有很大影响,有可能算法比较慢,但是实际影响并不大.

常见名词

  • 时间复杂度: 指算法需要消耗的时间资源.一般量化为操作次数.
  • 空间复杂度: 指算法需要消耗的空间资源.一般量化为需要的交换空间等.
  • 稳定性: 如果a原本在b前面,而a=b,排序之后a仍然在b的前面就是稳定的;反之是不稳定的.
  • 内部排序: 所有操作都在内存中完成.如果需要结合外部存储配合就称为外部排序(比如数据量太大).

常见时间复杂度说明

Big-O 名字 描述
O(1) 常数级 最好的。不论输入数据量有多大，这个算法的运行时间总是一样的。
例子: 基于索引取出数组中对应的元素。
O(log2n) 对数级 相当好。这种算法每次循环时会把需要处理的数据量减半。
即便数据量很大这种算法也非常快。
例子：二分查找。如果你有 100 个元素，则只需要七步就可以找到答案。
O(n) 线性级 还不错。如果你有 100 个元素，这种算法就要做 100 次工作。
数据量翻倍那么运行时间也翻倍。
例子：线性查找。
O(n logn) 线性对数级 还可以。比线性级差了一些，不过也没那么差劲。
例子：最快的通用排序算法。
O(n^2) 二次方级 有点慢。如果你有 100 个元素，这种算法需要做 100^2 = 10000 次工作。
数据量 x 2 会导致运行时间 x 4 (因为 2 的 2 次方等于 4)。
例子：循环套循环的算法，比如插入排序。
O(n^3) 三次方级 特别慢。如果你有 100 个元素，那么这种算法就要做 100^3 = 100,0000 次工作。
数据量 x 2 会导致运行时间 x 8。
例子：矩阵乘法。
O(2^n) 指数级 超级慢。这种算法你要想方设法避免，但有时候你就是没得选。
加一点点数据就会把运行时间成倍的加长。
例子：旅行商问题。
O(n!) 阶乘级 比蜗牛还慢！不管干什么都要跑个 N 年才能得到结果。

HTTP协议

发表于 2019-08-26 | 更新于 2019-09-30 | 分类于 http

HTTP是最常见的应用层协议,也是WEB开发不可避免的要熟悉的知识.

HTTP简介

HTTP(超文本传输协议)是在网络上传输HTML的协议,用于浏览器和服务器之间的通信,由请求和响应组成,是一个标准的CS模型.同时HTTP是一个无状态协议.

HTTP协议通常承载于TCP协议之上,有时也承载于TLS或SSL协议层之上,这个时候HTTP就成了我们常说的HTTPS.

HTTP最广泛使用的版本是1.1;目前最新的版本是2.0.之前还有0.9,1.0.

HTTP 1.1相对于1.0最主要的变化是支持长连接(Persistent Connection:建立一次 TCP 连接就能进行多次 HTTP 通信)和请求的流水线(Pipelining:在同一条长连接上连续发出请求,而不用等待响应返回,这样可以减少延迟)处理.在一个TCP连接上可以传送多个HTTP请求和响应.减少了建立和关闭连接的消耗和延迟.在HTTP1.1中默认开启Connection:keep-alive.

HTTP2.0多路复用(多个请求可同时在一个连接上并行执行)相对于HTTP1.x的管道流水线(若干个请求排队串行化处理)来说,性能更好.

OSI (Open System Interconnection)七层模型

OSI 开放系统互联模型,描述的是网络或者电信系统功能的概念性框架.分为 7 层(自底向上):

  1. 物理层(Physical): 比特(电信号,光信号)在不同介质中的表示.具体设备:网线.
  2. 数据链路层(Data Link): 负责两个直接连接的节点之间的数据传输,以及针对物理层的纠错.分为MAC 层(媒体访问控制)和 LLC层(逻辑链路控制).具体设备:交换机.
  3. 网络层(Network): 通过不同路由分组转发.具体设备:路由器. IP 协议.
  4. 传输层(Transport): 负责传输数据以及可能的算法纠错等. TCP, UDP 协议.
  5. 会话层(Session): 建立,管理,维护会话;简化传输层操作.
  6. 表示层(Presentation): 负责数据的应用格式和网络格式之间的转换.比如加解密.
  7. 应用层(Application): 负责建立与用户直接交互的应用程序.如浏览器.
