SpringBoot微服务实践

发表于 2019-07-04 | 分类于 微服务 | 阅读次数：

最近学习了如何基于SpringBoot开发微服务.记录备忘.

学习资料:

<<[Moises_Macero]_Learn_Microservices_with_SpringBoot.pdf>>

社区 Spring Cloud 从入门到进阶系列教程

整体架构:

模块列表:

  • UI服务器: 负责所有的静态资源
  • 服务注册中心: 所有的业务模块都要注册到该中心,要实现高可用
  • Gateway: 负责对外的统一入口,路由各个服务
  • Multiplication, Gamification: 具体业务模块,可以很容易的水平扩展
  • RabbitMQ: 消息总线,负责各个业务模块之间通信
  • Config: 配置中心,实现各个业务模块的配置版本控制管理

问题:

  1. 各个模块从配置中心获取配置的属性相同,可以制作一个模板,但是容易复制粘贴出错.
  2. 服务发现中心对于服务的可用性发现有一定的延迟,会有短暂的服务不可用.需要注意容错.
  3. 注册中心的地址hardcode在各个模块的配置中,容易出错.
  4. 如何把所有模块的HystrixDashboard融合在一起?单个集群可以使用Turbine聚合.

每一个组件还有很多细节需要探究明白,边做边看.代码放在github留存.
