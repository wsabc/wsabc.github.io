基础数据结构

发表于 2019-09-25 | 更新于 2019-10-10 | 分类于 data-structure | 阅读次数：

学习数据结构,是解决业务问题和学习算法必不可少的环节.

数组和字符串

数组和字符串是最基本的数据结构,这部分的算法面试题也是最多的.

数组的优缺点

优点 缺点
构建一个数组非常简单 构建时必须分配一段连续的空间
可以根据数组的下标(index)查询某个元素,时间O(1) 查询某个元素是否存在时要遍历整个数组,时间O(n)
删除/添加某个元素时需要移动剩余元素,时间O(n)

在考虑是否采用数组解决问题时,必须要考虑它的优缺点,看看它的缺点是否会阻碍所用算法的时间复杂度以及空间复杂度.比如很多时候字符串问题就要先把给定的字符串转换成字符数组之后再进行分析和处理.

例子: 翻转一个字符串.

一种比较快速和直观的方法是用两个指针,一个指向字符串的第一个字符,一个指向它的最后一个字符,然后互相交换.交换之后,两个指针向中央一步步地靠拢并相互交换字符,直到两个指针相遇(或者头>尾).由于无法直接修改字符串里的字符,所以必须先把字符串变换为数组,然后再运用这个算法.

练习: 力扣第242题. Valid Anagram 判断两个字符串是否互为字谜. 提示: 利用数组统计字母出现次数并比较.

链表

链表和数组是互补的结构.主要是内存分配不一样,以及由此带来的特性.第一个节点称为head,最后一个节点称为tail.Tail总是指向null.通常的实现有2种:没有虚拟头节点和有虚拟头节点(更常用,实现简单).

链表的优缺点

优点 缺点
无需连续的内存空间 查询第i个元素需要O(i)时间
在知道前一个元素的情况下,删除/添加元素快,时间O(1)
有单向链表,双向链表之分,可能有环.  

一般来说,如果数据的元素个数不确定,而且需要经常进行数据的添加和删除,那么链表会比较合适;而如果数据元素大小确定,删除插入的操作并不多,或者快速定位元素的操作多,那么数组可能更适合.

常见的使用链表的方法有: 使用快慢指针; 利用虚拟的链表头等.在解决链表问题时画示意图可以帮助分析问题.

例子: 翻转单向链表.

使用递归或者循环.递归思想是将头节点的下一个节点当作一个新的链表对待.循环思路是pre指向null,head,tmp指向head.next.往后平移head变成pre,tmp变成head.

例子: 判断链表是否有环.

使用快慢指针,一旦两个指针重合,说明有环.(Floyd Cycle Detection环检测算法).

栈

栈的特点就是后进先出,对于栈中的数据来说,所有操作都是在栈的顶部完成的(入栈,出栈,查询).

可以利用一个单链表来实现栈的数据结构(单链表头插法),借用单链表的头就能让所有栈的操作在O(1)的时间内完成.也可以用一个数组外加一个指针(位置指针)也能实现相似的效果,但是一旦数组的长度发生了改变,哪怕只是在最后添加一个新的元素,时间复杂度就不再是O(1).

当解决某个问题的时候只关心最近一次的操作,并且在操作完成了之后,需要向前查找到更前一次的操作.这种场合就适合使用栈.

例子: 括号正确匹配.

利用栈,所有左括号入栈,当遇到右括号时出栈,并比较是否匹配.

队列

队列的特点是先进先出,就像排队一样.队头队尾都允许查看,但是只允许在队尾添加数据,在队头删除数据.

当需要按照一定的顺序来处理数据,而要处理的数据量在不断地变化的时候,就需要使用队列.

双端队列: 双端队列和普通队列最大的不同在于,它允许在队列的头尾两端都能在O(1)的时间内进行数据的查看/添加和删除.与队列相似,可以利用一个双链表来实现双端队列.双端队列最常用的地方就是实现一个长度动态变化的窗口或者连续区间.

例子: 用单链表实现一个队列.

一个front一个rear相当于单链表里的head和tail.

树

树的结构十分直观,而树的很多概念定义都有一个相同的特点:递归,也就是说,一棵树要满足某种性质,往往要求每个节点都必须满足.例如,在定义一棵二叉搜索树时,每个节点也都必须是一棵二叉搜索树.

需要熟悉的有普通二叉树,平衡二叉树,完全二叉树,二叉搜索树,四叉树(Quadtree),多叉树(N-ary Tree).

对于一些特殊的树例如红黑树(Red-Black Tree),自平衡二叉搜索树(AVL Tree)了解即可.

树的基本遍历有三种:前序遍历(Preorder Traversal), 中序遍历(Inorder Traversal)以及后序遍历(Postorder Traversal).

前 中 后 这三个词是针对根节点的访问顺序而言的,即前序就是根节点在最前根->左->右,中序是根节点在中间左->根->右,后序是根节点在最后左->右->根.
