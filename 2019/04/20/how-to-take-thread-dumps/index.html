How to take Thread Dumps

CPU spikes, deadlocks, poor response times, memory problems, unresponsive applications, .etc

  1. jstack to print Java stack traces of Java threads for a given Java process.

jstack是从Java5开始提供的工具。

  1. Kill -3适用于没有JDK(jstack是JDK工具包里的)的环境

kill -3会将dump文件发送到标准错误输出，如果是tomcat，会在catalina.out文件里

  1. jvisualvm

JDK6 update 7开始提供的GUI工具。可以dump远程jvm。

  1. JCMD

jcmd是JDK7开始提供的，要求必须在jvm运行的机器上执行。除了thread dump，还可以输出heap dump。

jstack和jcmd适合有JDK的环境，kill -3适合没有JDK的环境

具体诊断顺序

  1. top -H
  2. 捕获Thread dump
  3. 查找对应pid(转成16进制)
  4. 或者使用 https://fastthread.io/ 分析
