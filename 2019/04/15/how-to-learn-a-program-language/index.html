如何掌握一门编程语言

发表于 2019-04-15 | 分类于 program | 阅读次数：

看完王垠的文章 <<如何掌握所有的程序语言>> 有所启发,记录如下.

透过现象看本质

重视语言特性,而不是语言(品牌).

任何一种语言,都是一堆语言特性的集合,就好比品牌电脑,肯定是由一堆通用组件,然后可能会加上自己的特色组件组成.我们更多关注的应该是语言特性.

  • 变量定义
  • 算术运算
  • 控制结构
  • 函数定义, 函数调用
  • 递归
  • 静态类型系统
  • 类型推导
  • lambda函数
  • 面向对象
  • 垃圾回收
  • 指针算术
  • goto 语句

如果用不好现在的语言,不要指望换一种语言就能让你写出优雅的代码.

学习从解决问题开始

常见的学习语言的方式都是从一本语言说明书开始.这样容易失去关注点,在次要特性上投入较多的注意力.较好的方式是带着问题入手,在解决问题的过程中熟悉语言的主要特性.

在这个过程中, 随着需求的出现, 他可能会问这样的问题:

  • 这个语言的“变量定义”是什么语法, 需要“声明类型”吗, 还是可以用“类型推导”?
  • 它的“类型”是什么语法? 是否支持“泛型”? 泛型的 “variance” 如何表达?
  • 这个语言的“函数”是什么语法, “函数调用”是什么语法, 可否使用“缺省参数”?
  • ……

分清主要矛盾和次要矛盾

主要矛盾很少,次要矛盾很多.如果关注点在次要矛盾上,那将是无穷无尽的烦恼,也不要奢望能解决所有的次要矛盾.

先解决行不行,再看好不好.

绝知此事要躬行

完全理解一种“语言特性”最好的方法就是自己亲自实现它.
