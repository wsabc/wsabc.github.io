编程技巧

发表于 2019-04-15 | 分类于 program | 阅读次数：

读书笔记, 来自于王垠的<<编程的智慧>>一文.

世界上有两种“没有bug”的代码: 一种是“没有明显的bug的代码”,另一种是“明显没有bug的代码”.怎么写出明显没有bug的代码呢?

  • 反复推敲代码

    这个就像练字,不是看你写了多少字,而是看你每个字写的质量如何.有的字刚写完,觉得完美.过几日再看看,就觉得不如当时好看,有改进的空间.写代码也是如此.

  • 写优雅的代码

    优雅的代码都有一定的外貌特征:结构整齐,分明.就好像收拾干净的房间,没有多余的东西,需要用的东西也是分门别类摆放,用的时候立刻就能找到.
    又好比说英语,有人能用简单常见的单词来表达复杂的意思,有人堆砌一堆复杂的单词,却也表达不清楚自己的意图.

  • 写易读的,简单的代码

    • 避免太长的函数
    • 利用或者制造助手函数,即使只有两行,也能简化主函数里的逻辑
    • 每个函数只做一件事情
    • 避免使用全局变量和类成员来传递信息,尽量使用局部变量和参数
    • 使用有意义的函数名字和变量名字
    • 局部变量尽量靠近使用它的地方,这样局部变量的名字也可以简短一点,尽量不要对局部变量进行多次赋值以免增加理解障碍
    • 不要书写过长的,复杂的表达式
    • 保持合理换行
    • 合理使用括号,不要省略花括号
    • 避免使用continue和break,尽量避免使用自增自减表达式
    • 写直观的代码,保持简单,不要让读代码的人费力思考

  • 写”无懈可击”的代码

    • 不要放弃任何一个分支.你最终是需要考虑这种情况的.覆盖每一个分支,也让读代码的人头脑开销少点.
    • 分开处理异常,细粒度处理异常.
    • 尽量不要产生Null,也不要让Null无限制流转,函数作者应该限制Null进来,并提醒调用者Null产生的可能性.调用者应该及时处理Null而不是抛出去.

  • 防止过度工程(最难杜绝的”提早优化”)

    过度工程即将出现的一个重要信号,就是当你过度的思考“将来”,考虑一些还没有发生的事情,还没有出现的需求.
    另外一种过度工程的来源,是过度的关心“代码重用”,过度关心”测试”.

    • 先把眼前的问题解决掉,解决好,再考虑将来的扩展问题.
    • 先写出可用的代码,反复推敲,再考虑是否需要重用的问题.
    • 先写出可用,简单,明显没有bug的代码,再考虑测试的问题.
